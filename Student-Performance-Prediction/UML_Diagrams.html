Class Diagram

Student Performance Prediction System
System Classes: The User entity manages authentication (Admin, Teacher, Student roles). The Student entity holds academic records and is composed of TheoryMarks and PracticalMarks value objects. The PredictionEngine service (Python FastAPI) applies ML to compute risk level, AI insights, and performance score. The AuthService handles JWT token lifecycle.

Diagram 05

State Chart Diagram

Student Performance Prediction System
States: The system transitions through: Unauthenticated → Authenticating → Authenticated. From Authenticated, users navigate to Student List, Add/Edit forms, and Detail views. Adding a student automatically transitions to Predicting state where the AI engine runs. Errors transition to an Error state with retry logic.

Student Performance Prediction System
Data Flows: External entities (Admin/Teacher and Student) drive the system. Process 1.0 handles authentication reading/writing to Users Collection. Entity-Relationship (ER) Diagram

Student Performance Prediction System
ER Summary: The USER entity manages system access (1:N manages Students). Each STUDENT embeds a PREDICTION sub-document (1:1) containing AI results — avoiding a separate collection for simplicity. The SESSION is stateless (JWT), logically linked to USER with a 1:N relationship.
